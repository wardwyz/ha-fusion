Approcci per la Review Grafica Modali

101 file di modali, problemi di scroll + layout + spaziatura su più risoluzioni

A

Global-first

Fix del container globale in Index.svelte (max-height, overflow, breakpoint), poi revisione manuale delle modali più usate. Veloce, copre il caso comune ma può lasciare problemi specifici per modale.

Pro

  • Rapido
  • Fix immediato per tutti

Contro

  • Non copre problemi per-modale
  • Revisione incompleta
B

Modale per modale

Review sistematica di tutti i 101 file: screenshot, analisi, fix. Massima copertura ma enormemente lento. Non pratico per una singola sessione.

Pro

  • Copertura completa
  • Nessun problema lasciato

Contro

  • 101 file = molto lento
  • Molte modali non hanno problemi
C

Per categorie — Raccomandato

Fix globale del container + revisione per categoria: Config modali (edit mode), Entity detail modali (interazione entità), Modali speciali (AI, Camera, PictureElements, Calendar). Bilanciato tra copertura e velocità.

Pro

  • Strutturato e completo
  • Fix sistemico + per-caso
  • Gestibile in una sessione

Contro

  • Richiede prioritizzazione

Dettaglio approccio C (categorie)

Fase 1 — Globale
  • Index.svelte: max-height, overflow, breakpoint
  • Modal.css: stili condivisi
Fase 2 — Config modali (edit mode)
  • ButtonConfig, LightConfig, ClimateConfig…
  • SidebarItemConfig, MainItemConfig
  • VisibilityConfig
Fase 3 — Entity detail modali
  • LightModal, ClimateModal, MediaPlayer…
  • Tutti i ~43 modal di entità
Fase 4 — Modali speciali
  • AiAssistantModal (60vh custom)
  • PictureElementsConfig (canvas editor)
  • CalendarModal, DoorbellModal
